 tcp/ip 如果两个URL一样，每次都连接， 断开开销大
 - keep-alive 允许在同一个tcp连接上 传输多个http请求和响应，减少了连接的建立和关闭次数
 - 管道化(piplelining): 允许客户端在同一个连接上面发送多个请求，而无需等待每个请求响应。这可以减少延迟，提高性能。

 多路复用

 服务器推送 server push
 服务器可以主动向客户端发送资源，而无需客户端请求(当客户端发送请求，请求1时，服务器知道还需要请求2时，
